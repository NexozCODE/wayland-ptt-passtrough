# PTT Passthrough (Hyprland/Wayland → Discord)

Перехватывает кнопки мыши на уровне evdev и шлёт их в Discord как клавиатурные клавиши через XWayland, минуя ограничение Wayland на глобальные хоткеи/кнопки мыши без фокуса окна.

## Проблема, которую решает

- Discord (официальный клиент, pacman) рендерится через XWayland.
- Discord слушает ввод только через X11-слой → события чистого Wayland (Hyprland native) до него не доходят без фокуса.
- Поэтому кнопки мыши, забинженные в Discord (например "Mouse 9"), не работают глобально под Hyprland/Wayland — только когда окно Discord в фокусе.
- Решение: ловим нужную кнопку через evdev (в обход композитора), грабим устройство, и инжектим соответствующую клавиатурную клавишу прямо в X-сервер через `xdotool` — Discord видит её как обычное системное нажатие клавиатуры, независимо от фокуса.
- Все остальные события мыши (движение, ЛКМ/ПКМ, скролл) форвардятся как есть через виртуальное uinput-устройство, чтобы мышь продолжала нормально работать в остальной системе.

## Зависимости

```bash
sudo pacman -S python-evdev xdotool
```

Права на устройства ввода и virtual device:

```bash
sudo usermod -aG input $USER
echo 'KERNEL=="uinput", GROUP="input", MODE="0660"' | sudo tee /etc/udev/rules.d/99-uinput.rules
sudo udevadm control --reload-rules && sudo udevadm trigger
# перелогиниться после usermod, иначе группа не подхватится
```

## Использование

### 1. Настройка биндов (один раз, или при смене мыши/добавлении новой кнопки)

```bash
python3 ptt_passthrough.py --discover
```

Скрипт попросит нажать кнопку — определит устройство и код автоматически (без `evtest`). Дальше спросит:
- какую клавишу слать (например `F13`, `F14` — что-то отсутствующее на обычной клавиатуре, чтобы не нажималось случайно);
- режим: `hold` (зажал/отжал — для PTT) или `toggle` (одно нажатие — для mute/deafen и подобного).

Можно добавить несколько биндов подряд в одной сессии discovery.

Конфиг сохраняется в `~/.config/ptt-passthrough/config.json`:

```json
{
  "bindings": [
    {"device_name": "Compx 2.4G Wireless Receiver", "button_code": 276, "key": "F13", "mode": "hold"}
  ]
}
```

### 2. Настройка Discord

В Настройки → Голос и видео → Push to Talk Keybind → Edit Keybind → нажать ту клавишу, что указали в discovery (например `F13`). Аналогично для mute/deafen, если добавляли toggle-бинд.

### 3. Запуск

```bash
python3 ptt_passthrough.py
```

Слушает все настроенные кнопки, форвардит их в Discord через xdotool, остальной ввод мыши идёт прозрачно. Работает независимо от фокуса окна.

## Автозапуск с Hyprland (опционально)

В `~/.config/hypr/hyprland.conf`:

```
exec-once = python3 /путь/до/ptt_passthrough.py
```

XWayland может быть не готов в момент старта сессии — если бинды не подхватываются сразу при логине, добавить задержку:

```
exec-once = sh -c 'sleep 2 && python3 /путь/до/ptt_passthrough.py'
```

## Поведение при смене мыши / переключении режимов

- При запуске без `--discover` скрипт ищет устройство сначала по точному имени из конфига, а если не находит — по совпадению кода кнопки на любом подключённом устройстве (fallback). Это позволяет безопасно сменить мышь в будущем — старый конфиг продолжит работать, если новая мышь имеет кнопку с тем же кодом (для side-кнопок это почти всегда так).
- При смене проводной/беспроводной мыши на одном ресивере (или физическом отключении устройства) скрипт ловит `OSError` при потере устройства, пересканирует подключённые устройства и переподключается автоматически — без перезапуска.
- Если кнопки из разных биндов физически находятся на разных устройствах одновременно (например PTT на мыши + mute на отдельной макро-клавиатуре) — текущая версия обрабатывает только первое найденное устройство за раз. Для одновременной поддержки нескольких физических устройств нужна доработка (расширение run_session на несколько fd через select, аналогично discovery-режиму).

## Известные ограничения

- Работает только с официальным Discord-клиентом (через XWayland). Не работает с Vesktop — там обработка хоткеев полностью игнорируется без фокуса окна.
- Требует `xdotool` и доступного `$DISPLAY` (XWayland) в момент запуска.
- Полный grab устройства означает, что весь трафик мыши идёт только через этот скрипт — если скрипт упадёт без штатного `finally`/`ungrab`, мышь может временно "залипнуть" (требуется завершить процесс или перезапустить сессию).
